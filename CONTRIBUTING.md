# Contributing to PDF Batch Merger

Thank you for your interest in contributing. Please follow the project's coding style so reviews stay focused on behavior.

## Code style

**See [docs/STYLE.md](docs/STYLE.md)** for the full style guide. In short:

- **Quotes**: Use double quotes (`"`) for strings.
- **Docstrings**: Use [Google style](https://google.github.io/styleguide/pyguide.html#38-comments-and-docstrings) (Args, Returns, Raises) for the public API.
- **Trailing commas**: Use trailing commas in multi-line lists, dicts, and argument lists.
- **Boolean results**: Name boolean return values at call sites (e.g. `success = convert_excel_to_pdf(...)`).

New code should match the style of the existing codebase and the guidelines in `docs/STYLE.md`.
